Radix Sort

Radix Sort is a non-comparative sorting algorithm that processes numbers digit by digit, from the least significant to the most significant. It is highly efficient for sorting large numbers with a fixed range but requires extra space for intermediate sorting steps.


Visual Representation


Explanation

Radix Sort works by sorting numbers based on their individual digits, moving from the least significant digit (LSD) to the most significant digit (MSD). Instead of comparing values directly, it groups numbers into "buckets" based on each digit's value.

Steps of Radix Sort:

  1. Identify the maximum number in the array to determine the number of digits (d).
  2. Start sorting based on the least significant digit (LSD) using a stable sorting algorithm (often Counting Sort).
  3. Move to the next digit and repeat the sorting process.
  4. Continue until all digits have been processed.
  5. The array is now sorted.

How Radix Sort Works?

Radix Sort follows the LSD (Least Significant Digit) approach, which sorts numbers by their digits from right to left.

Example

Consider the following array:

[720, 329, 839, 457, 657, 355, 436]

Sorting process (Step-by-step):

                        Step 1: Sort by least significant digit (units place)
                        [720, 329, 839, 457, 657, 355, 436]
                        Step 2: Sort by tens place 
                        [720, 329, 436, 839, 355, 457, 657]
                        Step 3: Sort by hundreds place  
                        [329, 355, 436, 457, 657, 720, 839]

The array is now fully sorted.


Time Complexity

Best Case

O(nk)

Average Case

O(nk)

Worst Case

O(nk).

  • n = Number of elements
  • k = Number of digits in the largest number

Radix Sort performs well when k (the number of digits) is small but may become inefficient when k is large.

Space Complexity

  • O(n + k) (Extra space is needed for intermediate bucket sorting)
