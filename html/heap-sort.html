Heap Sort

Heap Sort is a comparison-based sorting algorithm that builds a binary heap and extracts elements in sorted order. It is efficient with a time complexity of O(n log n) for all cases, making it more optimal than algorithms like Bubble Sort and Insertion Sort for large datasets.

Explanation

Heap Sort is a comparison-based sorting algorithm that utilizes a binary heap data structure to sort elements. It operates by organizing the input data into a special tree-based structure called a heap, which satisfies the heap property.

In a max heap, for example, each parent node is greater than or equal to its child nodes, ensuring that the largest element is at the root. This property allows Heap Sort to efficiently extract the maximum element and place it in its correct position in the sorted array.


How Heap Sort Works?

  1. Build a Max Heap: Convert the unsorted input array into a max heap. This arrangement ensures that the largest element is at the root of the heap.
  2. Extract Maximum Element: Swap the root element (maximum value) with the last element of the heap. This action moves the largest unsorted element to its correct position in the array.
  3. Reduce Heap Size: Exclude the last element (now sorted) from the heap by reducing the heap size.
  4. Restore Heap Property: Heapify the root element to maintain the max heap property. This process ensures that the next largest element moves to the root.
  5. Repeat: Continue steps 2-4 until all elements are sorted and the heap is empty.
  6. Continue iterating until no swaps are needed in a complete pass.

Example

Consider an array:

[5, 3, 8, 4, 2]

Step-by-step Heap Sort process:

  1. Build Max Heap: Convert the array into a max heap. 
    [8, 3, 5, 4, 2]  (Heapified)
  2. Swap max element with last and reduce heap size: 
    [2, 3, 5, 4, 8]  (Swapped 8 with 2)
  3. Heapify root (restore heap property): 
    [5, 3, 2, 4, 8]
  4. Repeat the process: 
                                    [4, 3, 2, 5, 8]  (Swap 5 with 2)
                                [3, 2, 4, 5, 8]  (Heapify)
                                [2, 3, 4, 5, 8]  (Sorted)

Time Complexity

Best Case

O(n log n) - Balanced input

Average Case

O(n log n) - Random order

Worst Case

O(n log n) - Reverse order


Space Complexity

Heap Sort has an **in-place** implementation, meaning it requires **O(1)** additional space. However, in some recursive implementations, it may require O(log n) space due to recursion depth.
