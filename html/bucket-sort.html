Bucket Sort

Bucket Sort is a distribution-based sorting algorithm that distributes elements into multiple buckets, sorts each bucket individually, and then merges the results. It works best when input data is uniformly distributed.


Visual Representation


Explanation

Bucket Sort divides elements into a fixed number of buckets and sorts them separately, often using another sorting algorithm like Insertion Sort.

Steps of Bucket Sort:

  1. Distribute elements into several buckets based on value range.
  2. Sort each bucket using a suitable sorting algorithm (Insertion Sort, Merge Sort, etc.).
  3. Concatenate all sorted buckets to get the final sorted array.

How Bucket Sort Works?

Example

Consider an unsorted array:

[0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12, 0.23, 0.68]

Sorting process (Step-by-step):

Step 1: Distribute into Buckets
Assume 5 buckets for values between 0 and 1: 
                                Bucket 1: [0.12]  
                                Bucket 2: [0.17, 0.21, 0.23, 0.26]  
                                Bucket 3: [0.39]  
                                Bucket 4: [0.68, 0.72, 0.78]  
                                Bucket 5: [0.94]
Step 2: Sort Individual Buckets
Each bucket is sorted using Insertion Sort: 
                                Bucket 1: [0.12]  
                                Bucket 2: [0.17, 0.21, 0.23, 0.26]  
                                Bucket 3: [0.39]  
                                Bucket 4: [0.68, 0.72, 0.78]  
                                Bucket 5: [0.94]
Step 3: Concatenate Buckets
Final sorted array: 
                                [0.12, 0.17, 0.21, 0.23, 0.26, 0.39, 0.68, 0.72, 0.78, 0.94]

Time Complexity

Best Case

O(n + k) (If elements are evenly distributed)

Average Case

O(n + k)

Worst Case

O(nÂ²) (If all elements fall into the same bucket, behaving like Insertion Sort)

Where n is the number of elements and k is the number of buckets.


Space Complexity

  • O(n + k) (Requires additional space for buckets)

Bucket Sort is not in-place, as it requires extra storage.

A Thought on Bucket Sort...

"Bucket Sort is highly efficient when input data is evenly distributed over a known range. However, its performance degrades significantly when elements cluster into fewer buckets, effectively reducing it to quadratic time complexity. Choosing the right number of buckets and a suitable sorting method within each bucket is crucial for its success."
Thomas H. Cormen , Introduction to Algorithms (CLRS)
