Quick Sort

Quick Sort is a divide-and-conquer sorting algorithm that efficiently sorts an array by choosing a pivot, partitioning elements, and recursively sorting the subarrays. It is faster than many other sorting algorithms in practice due to in-place sorting and better cache performance.


Visual Representation


Explanation

Quick Sort is a divide-and-conquer sorting algorithm that works by selecting a pivot element and partitioning the array such that elements smaller than the pivot move to its left, and larger elements move to its right. The process is then recursively applied to both partitions until the entire array is sorted.

  • It is faster than Merge Sort in most practical cases due to lower constant factors.
  • It is an in-place sorting algorithm, meaning it does not require additional memory like Merge Sort.
  • However, its worst-case time complexity is O(n²) if the pivot selection is poor (e.g., always choosing the largest or smallest element).

How Quick Sort Works?

  1. Choose a Pivot (e.g., the last element, first element, median, or random element).
  2. Partition the Array: Rearrange elements so that elements smaller than the pivot are on the left and greater ones are on the right.
  3. Recursively Apply Quick Sort to the left and right subarrays.
  4. Continue Until the Array is Fully Sorted.

Example

Consider an array:

[5, 3, 8, 4, 2]

Step-by-step Sorting process:

                        Step 1: Choose pivot (e.g., 2) → Partition → [2, 3, 8, 4, 5]  
                        Step 2: Choose pivot (e.g., 3) → Partition → [2, 3, 8, 4, 5]  
                        Step 3: Choose pivot (e.g., 8) → Partition → [2, 3, 4, 5, 8]  
                        Step 4: Sorted → [2, 3, 4, 5, 8]

Time Complexity

Best Case

O(n log n) (when the pivot divides the array into balanced halves).

Average Case

O(n log n).

Worst Case

O(n²) (when the pivot is the smallest or largest element in an already sorted or reverse-sorted array).

Unlike Merge Sort, Quick Sort is generally faster in real-world scenarios due to better cache locality and fewer memory allocations.


Space Complexity

  • O(log n) (Best Case, Tail Recursion Optimization)
  • O(n) (Worst Case, Unbalanced Partitions)

Since Quick Sort is in-place, it does not require extra memory, unlike Merge Sort.
