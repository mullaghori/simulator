Selection Sort

Selection Sort is a simple comparison-based sorting algorithm that repeatedly finds the smallest element and places it in its correct position. Though inefficient for large datasets, it is easy to implement and requires minimal swaps.


Visual Representation


Explanation

Selection Sort works by repeatedly selecting the smallest (or largest) element from the unsorted portion of the array and moving it to its correct position.

Steps of Selection Sort:

  1. Start with the first element and assume it is the smallest.
  2. Compare it with the rest of the elements to find the actual smallest.
  3. Swap the smallest element with the first element.
  4. Move to the next element and repeat the process for the remaining unsorted portion.
  5. Continue until the entire array is sorted.

How Selection Sort Works?

Example

Consider an unsorted array:

[64, 25, 12, 22, 11]

Sorting process (Step-by-step):

                        Pass 1: (Find the smallest and swap)
                        [11, 25, 12, 22, 64]  (Swapped 11 and 64)
                        Pass 2: 
                        [11, 12, 25, 22, 64]  (Swapped 12 and 25)
                        Pass 3:  
                        [11, 12, 22, 25, 64]  (Swapped 22 and 25)
                        Pass 4:  
                        [11, 12, 22, 25, 64]  (No change, already sorted)

The array is now fully sorted.


Time Complexity

Best Case

O(n²) (Even if the array is sorted, it still performs comparisons)

Average Case

O(n²)

Worst Case

O(n²)

Selection Sort is not efficient for large datasets.


Space Complexity

  • O(1) (Constant space as sorting is done in-place)

Unlike Bubble Sort, Selection Sort minimizes the number of swaps but still performs O(n²) comparisons.
