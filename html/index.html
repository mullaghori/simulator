Different Categories

Comparison-Based Sorting

Sorting algorithms that compare elements of the dataset to determine their order. These algorithms rely on comparing pairs of elements using relational operators (e.g., <, >, =). Examples include Bubble Sort, Quick Sort, Merge Sort, and Heap Sort. Their time complexity is generally lower-bounded by O(n log n) for comparison-based sorting.

Non-Comparison Based Sorting

Sorting algorithms that do not compare elements directly but use other properties of the data (e.g., digits, counts) to sort. These algorithms are often faster than comparison-based sorts, with time complexities like O(n) or O(n + k). Examples include Counting Sort, Radix Sort, and Bucket Sort. They are useful for specific types of data (e.g., integers within a known range).

Other Sorting Categories

Sorting techniques that do not strictly fall into comparison or non-comparison categories includes hybrid algorithms (e.g., Timsort, which combines Merge Sort and Insertion Sort) and specialized algorithms for specific data structures (e.g., sorting networks) or otherwise;

In-Place/ Out-Place Technique
In-Place: Sorting algorithms that use a constant amount of extra memory (O(1) space) to rearrange elements are called In-Place Sorting Techniques. Examples include Bubble Sort, Insertion Sort, and Quick Sort. They are memory-efficient but may have higher time complexity.
Out-Place: While Out-Place Techniques includes Sorting algorithms that require additional memory proportional to the input size (O(n) space). Examples include Bubble Sort, Insertion Sort, and Quick Sort. They are memory-efficient but may have higher time complexity.

Online/ Offline Technique
Online: Sorting algorithms that can process input data as it is received, without needing the entire dataset upfront are called Online Sorting Techniques. Examples include Insertion Sort and Heap Sort. Useful for streaming data.
Offline: Offline Techniques can be defined as Sorting algorithms that require the entire dataset to be available before sorting begins. Examples include Merge Sort and Quick Sort. Suitable for static datasets.

Stable/ Unstable Technique
Stable: Stable Sorting algorithms are algorithms that maintain the relative order of equal elements. Examples include Merge Sort, Insertion Sort, and Bubble Sort. Useful when preserving the original order of equal elements is important.
Unstable: While on the other hand, Unstable Sorting algorithms are the ones that do not guarantee the relative order of equal elements. Examples include Quick Sort and Heap Sort. Faster but may not preserve order.

Different Algorithms

Bubble Sort

A simple comparison-based sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order.

Time Complexity: O(n²) (worst and average case).

Counting Sort

A non-comparison-based sorting algorithm that counts the occurrences of each element and uses arithmetic to place elements in the correct position.

Time Complexity: O(n + k), where k is the range of input.

Heap Sort

A comparison-based sorting algorithm that uses a binary heap data structure to sort elements.

Time Complexity: O(n log n).

Insertion Sort

A simple comparison-based sorting algorithm that builds the final sorted array one element at a time by inserting each element into its correct position.

Time Complexity: O(n²) (worst and average case), O(n) (best case for nearly sorted data).

Merge Sort

A divide-and-conquer, comparison-based sorting algorithm that recursively divides the dataset into smaller subarrays, sorts them, and merges them back together.

Time Complexity: O(n log n).

Quick Sort

A divide-and-conquer, comparison-based sorting algorithm that selects a pivot element and partitions the dataset into two subarrays (less than and greater than the pivot), then recursively sorts them.

Time Complexity: O(n log n) (average case), O(n²) (worst case for poorly chosen pivots).

Radix Sort

A non-comparison-based sorting algorithm that sorts numbers by processing individual digits (from least significant to most significant).

Time Complexity: O(nk), where k is the number of digits.

Selection Sort

A simple comparison-based sorting algorithm that repeatedly selects the smallest (or largest) element from the unsorted portion and swaps it with the first unsorted element.

Time Complexity: O(n²).

Bucket Sort

A non-comparison-based sorting algorithm that distributes elements into buckets and sorts each bucket individually.

Time Complexity: O(n + k), where k is the number of buckets.

Shell Sort

A generalization of Insertion Sort that allows exchanges of elements that are far apart.

Time Complexity: O(n log n) to O(n²) (depends on gap sequence).

Tim Sort

A hybrid sorting algorithm derived from Merge Sort and Insertion Sort, used in Python and Java.

Time Complexity: O(n log n).
