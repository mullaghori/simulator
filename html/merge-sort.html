Merge Sort

Merge Sort is a stable and efficient divide-and-conquer algorithm that recursively splits an array into smaller subarrays, sorts them, and then merges them back in a sorted order. It guarantees O(n log n) time complexity in all cases but requires additional memory for merging.

Explanation

Merge Sort is a divide-and-conquer sorting algorithm that recursively divides an array into smaller subarrays, sorts them, and then merges them back together.

  • The array is recursively split into two halves until each subarray contains only one element.
  • These subarrays are then merged back in sorted order.
  • The merging process ensures that the elements are placed in the correct order while combining the subarrays.

Since it always divides the input into equal halves, Merge Sort has a consistent time complexity of O(n log n), making it efficient for large datasets.


How Insertion Sort Works?

  1. Divide: Split the array into two halves.
  2. Recursively Sort: Apply Merge Sort on both halves.
  3. Merge: Combine the two sorted halves to form the final sorted array.

Example

Consider an array:

[5, 3, 8, 4, 2]

Step-by-step Sorting process:

                        Step 1: Divide → [5, 3, 8] and [4, 2]  
                        Step 2: Divide → [5, 3] and [8], [4] and [2]  
                        Step 3: Sort → [3, 5] and [8], [2, 4]  
                        Step 4: Merge → [3, 5, 8] and [2, 4]  
                        Step 5: Merge → [2, 3, 4, 5, 8] (Sorted)

Time Complexity

Best Case

O(n log n)

Average Case

O(n log n)

Worst Case

O(n log n)

Unlike algorithms like Quick Sort, Merge Sort always runs in O(n log n) time, even in the worst case.


Space Complexity

O(n) (Linear Space Complexity)

Unlike in-place sorting algorithms like Quick Sort or Heap Sort, Merge Sort requires extra space to store temporary subarrays.
