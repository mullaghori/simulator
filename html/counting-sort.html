Counting Sort

Counting Sort is a non-comparative sorting algorithm that sorts elements by counting occurrences. It works efficiently for small-range integers but struggles with large numbers or floating-point values.


Visual Representation


Explanation

Unlike comparison-based sorting algorithms (e.g., Bubble Sort, Quick Sort), Counting Sort follows a different approach by utilizing a counting array to determine the correct positions of elements in the sorted order.

Instead of repeatedly comparing elements, it counts the occurrences of each unique value within the given range and uses this frequency data to place elements in their correct sorted position.

The algorithm is efficient for sorting integers within a limited range, as it operates in linear time O(n + k), where n is the number of elements and k is the range of values.

However, its effectiveness declines when dealing with large numbers or when the range k is significantly larger than the number of elements.


How Counting Sort Works?

Steps:

  1. Find the range of input elements (smallest & largest value).
  2. Create a count array to store the frequency of each element.
  3. Modify the count array to store cumulative counts (helpful for placing elements).
  4. Place elements into the output array using the count array.
  5. Copy the sorted elements back to the original array.

Example

Sort the array 

[4, 2, 2, 8, 3, 3, 1]

Step-by-step process:

                        Find the Range: min = 1, max = 8
                        Initialize Count Array: [0, 1, 2, 2, 1, 0, 0, 0, 1] (size = max + 1)
                        Cumulative Count: [0, 1, 3, 5, 6, 6, 6, 6, 7]
                        Place Elements: [1, 2, 2, 3, 3, 4, 8] (sorted)

Note: Works well for small integers but inefficient for large numbers due to high space complexity.


Time Complexity

Best Case

O(n + k) (when k is small)

Average Case

O(n + k)

Worst Case

O(n + k) (large k increases memory usage)

Not suitable for very large k values, making it impractical for generic sorting.


Space Complexity

O(n + k) (Extra memory for count array)

Stable Sorting Algorithm (Preserves input order for identical elements)
